# Tier 1: native editable shapes in the designer

**For Sterling IT review.** This proposes a small, additive change to the
designer (`hteng/js/SMPdesigner.js`) so designs pushed from the Template
Generator arrive as **native editable vector shapes** instead of a single
flattened image, while richer effects (gradients, drop shadows, blends,
textures) are still flattened at high resolution.

## Why it's small and safe

The designer's production output is generated by **Fabric.js's own
`canvas.toSVG()`** (see `SMPdesigner.js` ~line 5908), and save/load use Fabric's
`toDatalessJSON` / `loadFromJSON`. So any object type Fabric already supports —
`polygon`, `path`, `ellipse`, `circle`, `rect`, `line` — flows through the
editor, save/load, **and the print pipeline automatically**, with no changes to
the ColdFusion production code.

Fabric already renders these shapes when loaded; they were simply not being
**categorised** into `canvas.shapeObjects`, so the shape tools didn't treat them
as first-class. The patch fixes only that.

## The change

See `DESIGNER_TIER1.patch` — **24 added lines, 0 removed**, entirely inside
`parseObjectsFromCanvas`:

1. An imported solid `circle` (not a non-printed marker) is pushed to
   `canvas.shapeObjects` (it previously fell through untracked).
2. A new branch catches `polygon` / `path` / `ellipse` / `triangle` / `line`
   and anything tagged `sterlingType: "shape"`, pushing it to
   `canvas.shapeObjects` (respecting the existing non-printed-object rule).

No existing behaviour changes; only previously-unhandled shape types gain
handling.

## What the generator now emits (see `generator/push-to-designer.js`)

Per element, in stacking order:

- **Text** → editable `i-text` (`sterlingType: "textObject"`).
- **Clean solid shapes** — rectangles, rounded rects, circles, ellipses,
  `clip-path` wedges (→ real **polygons**), quarter-discs — → native Fabric
  shapes (`sterlingType: "shape"`). Editable, recolourable, movable.
- **Everything else** — gradients, drop shadows, blends, textures, inline SVG
  ornaments — → one high-resolution (~300 dpi) flattened background image that
  sits behind the shapes and text. The flattening is `clip-path`- and
  `border-radius`-aware so clipped bands/wedges never fill their full box.

## Print-mode caveat

Multi-colour shapes and gradients apply to **Full Colour** products. Single
Colour, Greyscale, and Engraved Plastic products still require flattening or
colour conversion — the generator sets `designerVariationCode` accordingly and
IT's existing mode constraints continue to apply.

## Not yet included (next step)

Inline SVG ornaments (foliage, flourishes, icons) currently ride in the
flattened raster. Importing them as native Fabric vector groups via
`fabric.loadSVGFromString` is the recommended follow-up — same principle (Fabric
+ `toSVG` handle it), a bit more load-time plumbing.
